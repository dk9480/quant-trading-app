Methodology & Technical Decisions6.1. Data Handling & Persistence (SQLite)Choice: SQLite was chosen for its zero-configuration, file-based nature, ideal for a local prototype.Structure: Tick data is stored in a simple table (ticks) with columns for symbol, timestamp, price, and qty.Scaling Consideration: While sufficient for this prototype, for a high-frequency production system, this would be a major bottleneck. A robust production stack would replace this with a purpose-built time-series database (e.g., TimescaleDB, InfluxDB, or kdb+).6.2. Analytics and Decoupling (FastAPI)Analytics on Demand: The API handles all complex analytics on demand. This ensures the ingestion pipeline remains simple and fast (Write-Only).Code Reusability: Core quantitative logic (e.g., OLS calculation, Z-score) is encapsulated in a dedicated Python module, separate from the API routing (api.py), promoting clean design and testability.6.3. Frontend (React)Visualization: React was chosen for its component-based architecture, allowing for clean separation of concerns.User Experience: Provides interactive controls for selecting symbols, timeframe, lookback window, and regression type.7. Quantitative Analytics ImplementationThe core logic resides in the FastAPI endpoints:7.1. Price ResamplingRaw tick data is resampled to a regular time interval (e.g., 1-minute bars).Process: The API queries tick data and resamples to the specified timeframe (e.g., 1T).Aggregation: The last observed price (.last()) is used for the price series.7.2. Ordinary Least Squares (OLS) RegressionUsed to estimate the hedge ratio ($\beta$).Model: $\log(\text{S2}) = \alpha + \beta \cdot \log(\text{S1}) + \epsilon$Hedge Ratio ($\beta$): The slope coefficient ($\beta$) is calculated using the statsmodels.api library to perform a rolling OLS regression over the specified window.7.3. Spread/Residuals CalculationThe spread represents the deviation from the estimated long-term equilibrium.Calculation: $\text{Spread} = \log(\text{S2}) - (\beta \cdot \log(\text{S1}) + \alpha)$(Note: If OLS is run without the intercept, the formula simplifies to: $\text{Spread} = \log(\text{S2}) - \beta \cdot \log(\text{S1})$)7.4. Z-Score NormalizationNormalizes the spread to indicate standard deviation from the mean, serving as the signal.Calculation:$$\text{Z-Score} = \frac{\text{Spread} - \text{Rolling Mean}(\text{Spread})}{\text{Rolling Std Dev}(\text{Spread})}$$Signal Interpretation: $Z > 2.0$ signals Overbought (Short Spread), $Z < -2.0$ signals Oversold (Long Spread).7.5. Cointegration Testing (Augmented Dickey-Fuller - ADF)(If implemented) The ADF test is run on the spread series to confirm its stationarity (mean-reversion property).P-Value Interpretation: A P-value below 0.05 rejects the null hypothesis, confirming the pair is cointegrated.8.
